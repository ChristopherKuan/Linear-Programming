# 2.3 Linear Programs with Bounded Variables
<div align='justify'>
  Notice that in all the previous examples, all the constraints were not bounded, meaning that the constraints were like:<br><br>
  <div align='center'>
    x ≥ constant or x ≤ constant
  </div><br>
  Now we will look at bounded constraints:<br><br>
  <div align='center'>
    constant<sub>1</sub> ≤ x ≤ constant<sub>2</sub><br>
  </div><br>
  These constrains can be <b>handled with simplex method,</b> with the addition of slack and surplus variables, which handles the bounding constraints <b>explicitly.</b> However, this method makes the problem <b>more complicated and larger than necessary,</b> due to the introduction of variables.<br><br>
  Instead, it is better if the bounded-variable constraints were considered implicitly. In this approach, <b>pivoting calculations</b> are <b>only computed</b> for <b>equality constraints.</b>
</div>

## 2.3.1 Bounded Variable Optimality Condition
<div align='justify'>
  Before learning how to solve using implicit method, let us look at when is optimal solution achieved when having bounded variables. Consider the following <b>nonbasic variables</b> x<sub>1</sub> and x<sub>2</sub> constrained by:<br>
  <div align='center'>
    4 ≤ x<sub>1</sub> ≤ 15<br>
    2 ≤ x<sub>2</sub> ≤ 5
  </div><br>
  and the objective function
  <div align='center'>
    max z = 4 - 0.25x<sub>1</sub> + 0.5x<sub>2</sub>
  </div><br>
  Consider the following 2 cases:<br><br>
  <ol>
    <li>
      <b>x<sub>1</sub></b> at lower bound <b>(LB)</b> and <b>x<sub>2</sub></b> at upper bound <b>(UB)</b>
    </li><br>
    max z = 4 - 0.25(4) + 0.5(5) = 5.5<br><br>
    <li>
      <b>x<sub>1</sub></b> at <b>UB</b> and <b>x<sub>2</sub></b> at <b>LB</b>
    </li><br>
    max z = 4 - 0.25(15) + 0.5(2) = 1.25
  </ol>
  We can see that when the x with <b>negative coefficient</b> is at <b>LB</b> and x with <b>positive coefficient</b> is at <b>UB</b>, this gives the <b>max Z</b>, which is the optimal solution.<br><br>
  Based on this, we can understand the following condition:<br><br>
  <table align='center'>
    <tr align='justify'>
      <td>
        <div align='center'>
          <b>Bounded Variable Optimality Condition</b><br><br>
        </div>
        In a <b>maximization problem</b> in <b>canonical form,</b> if <b>every nonbasic</b> variable at its <b>lower bound</b> has a <b>nonpositive (negative) objective coefficient,</b> and every nonbasic variable at its <b>upper bound</b> has a <b>nonnegative (positive) objective coefficient,</b> then the basic feasible solution given by that canonical form <b>maximizes the objective function</b> over the feasible region.
      </td>
    </tr>
  </table>
  One question might arise: What if the <b>positive objective coefficient</b> has nonbasic variable x at <b>LB</b> and the <b>negative objective coefficient</b> has nonbasic variable x at <b>UB?</b><br><br>
  <ul>
    <li>We simply <b>increase</b> the value of x at <b>LB</b> and <b>decrease</b> the value of x at <b>UB.</b></li>
  </ul>
</div>

## 2.3.2 Implicit method for bounded variables
### Prerequisite knowledge
<div align='justify'>
  As mentioned before, we can change the value of nonbasic variable to LB or UB depending on its coefficient, and we wish to maintain feasibility when changing it. We must consider the following conteingencies:<br><br>
  <ol>
    <li>The <b>nonbasic variable</b> being changed reaches its <b>upper or lower bound</b>; or</li><br>
    In this case <b>no pivoting is required.</b> The nonbasic variable simply changes from its lower to upper bound, or upper to lower bound, and remains nonbasic.<br><br>
    <li>Some <b>basic variable</b> reaches either its <b>upper or lower bound.</b></li><br>
    In this case <b>pivoting</b> is used to <b>remove the basic variable</b> reaching either its lower or upper bound <b>from the basis.</b>
  </ol><br>
  Another thing to take note of is the <b>upper-bounding substitution</b>, which is done as follow:<br><br>
  <table align='center'>
    <tr>
      <td align='justify'>
        <div align='center'>
          Upper-bounding substitution
        </div><br>
        Whenever <b>x is nonbasic</b> at its <b>upper bound u,</b> the <b>slack variable x' = 0,</b> we <b>substitute x → u - x',</b> so that it makes x' nonbasic at value 0 in place of x.<br><br>
        Subsequently, when <b>x' becomes nonbasic at its upper bound</b> (also u), then <b>substitute x' → u - x,</b> so that it makes x appear nonbasic at value 0.
      </td>
    </tr>
  </table>
</div>

### Upper-bounding algorithm
<div align='justify'>
  Now, with the upper-bounding substitution knowledge, we can discuss the <b>upper-bounding algorithm.</b> This algorithm is applied to check the <b>incoming nonbasic variable during pivoting.</b> In this algorithm, we will compute three things:<br>
  <ol>
    <li>t<sub>1</sub></li>
    This is the usual ratio test.<br><br>
    <li>t<sub>2</sub></li>
    Perform such a check in <b>every constraint</b> in which the <b>incoming nonbasic variable</b> has a <b>negative coefficient.</b> The check is:<br><br>
    <table align='center'>
      <tr>
        <td>
          $$t_2 = \min_i \left\{ \left| \frac{u_k - b_i}{-a_{is}} \right| \;\middle|\; a_{is} < 0 \right\}$$
        </td>
      </tr>
    </table>
    Where <b>u<sub>k</sub></b> is the <b>upper bound</b> for the <b>basic variable x<sub>k</sub></b> in the <b>i<sub>th</sub> constraint</b>. <b>b<sub>i</sub></b> is the <b>current value for this variable,</b> and <b>a<sub>is</sub></b> is the <b>constraint coefficient</b> of the <b>incoming nonbasic variable x<sub>s</sub> in i<sub>th</sub> constraint.</b><br><br>
    <li>u<sub>s</sub></li>
    This is the upper bound of the current incoming nonbasic variable.
  </ol><br>
  The <b>incoming nonbasic variable x<sub>s</sub></b> will be set according to:<br><br>
    <div align='center'>
      x<sub>s</sub> = min {u<sub>s</sub>, t<sub>1</sub>, t<sub>2</sub>},
    </div>
  and when the minimum is:<br>
  <ol>
    <li>t<sub>1</sub></li>
    It means a <b>basic variable x<sub>k</sub> becomes 0,</b> so a usual <b>simplex pivot</b> is made to introduce the incoming nonbasic variable x<sub>s</sub> into the basis.<br><br>
    <li>t<sub>2</sub></li>
    It means some <b>basic variable x<sub>k</sub> reached</b> its <b>upper bound</b>, so perform <b>upper bounding substitution for x<sub>k</sub>.</b> Then use <b>simplex pivot on x<sub>k</sub></b> to <b>introduce x<sub>s</sub>.</b><br><br>
    <li>u<sub>s</sub></li>
    It means the <b>incoming nonbasic variable x<sub>s</sub></b> had <b>reached its upper bound,</b> so perform <b>upper bounding substitution</b> on <b>x<sub>s</sub>.</b>
  </ol><br>
  Let us look at an example to better understand it.
</div>

## 2.3.3 Conclusion
<div align='justify'>
  Linear programs with bounded variables can be solved using the simplex method either by explicitly adding constraints or by handling bounds implicitly, with the latter being more efficient as it avoids increasing problem size. The implicit bounded-variable method extends the simplex algorithm by allowing variables to move between their lower and upper bounds while maintaining feasibility. Optimality is achieved when all nonbasic variables at their bounds have reduced costs that do not allow further improvement of the objective function. The method introduces additional checks through the upper-bounding algorithm, where values t<sub>1</sub>, t<sub>2</sub>, and the upper bound determine whether a pivot occurs, a substitution is performed, or a variable simply changes bounds. Overall, this approach preserves the efficiency of the simplex method while correctly handling variable bounds without unnecessary complexity.
</div>
