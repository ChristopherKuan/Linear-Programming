# 2.2 Simplex method
<div align='justify'>
  We are finally at the stage to learn how to solve linear programming using the simplex method.<br><br>
  Consider the following form of equations:<br><br>

      </td>

    </tr>

  where <b>a</b> is the <b>constraint coefficients</b>, <b>b</b> is the <b>RHS value of the constraint</b>, <b>c</b> is the <b>coefficient of the objective function</b> and <b>z<sub>0</sub></b> is the <b>RHS value of the objective function</b><br><br>
  The <b>simplex algorithm</b> goes as follow:<br>
  <ol>
    <li>The problem is initially in canonical form and <b>all b ≥ 0.</b></li>
    <li>If <b>c<sub>j</sub> ≤ 0 for j = 1, 2, ..., n,</b> (all objective coefficient ≤ 0) then stop; we are at <b>optimal</b> The <b>optimal values</b> will be the <b>basic variables along with their current values</b>. Else continue</li>
    <li>Choose the <b>column to pivot in</b> (variable to introduce into basis) by choosing the column with the <b>largest positive coefficient in objective function.</b> Do take note that <b>some constraints within that column must be > 0,</b> or else stop as the primal problem is unbounded.
    </li>
    <li>Choose the <b>row to pivot in</b> (variable to drop from basis) using the <b>ratio test:</b></li><br>
    <table align='center'>
      <td>
      min { $\frac{b}{a}$ for whole column }</td>
    </table>
    <li><b>Replace the basic variable</b> in row with new basic variable and re-establish the canonical form.</li>
    <li>Go to step 1.</li>
  </ol>
  Note that these steps <b>apply to both phase 1 and phase 2 problem.</b> In <b>phase 1,</b> the equations that is referred to in step 3 and 4 is the <b>w equation.</b> In <b>phase 2,</b> it is the <b>objective function.</b><br><br>
  The <b>tableau that is used in the simplex method</b> has the following form:<br><br>

  Based on the headers, <b>put in all the coefficients and current values</b> for all constraints, objective function and w equation (only for phase 1), <b>then apply the 6 steps</b> to solve using simplex method.<br><br>

### Example (from 2.1.3 Phase 1-phase 2 procedure)
<div align='justify'>
  Consider the following linear program:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        <div align='center'>
          Maximize z = -3x<sub>1</sub> + 3x<sub>2</sub> + 2x<sub>3</sub> - 2x<sub>4</sub> - x<sub>5</sub> + 4x<sub>6</sub>,
        </div>
        <div align='left'>
          Subject to:
        </div>
        <div align='center'>
          x<sub>1</sub> - x<sub>2</sub> + x<sub>3</sub> - x<sub>4</sub> - 4x<sub>5</sub> + 2x<sub>6</sub> - x<sub>7</sub> + x<sub>9</sub> = 4 (Constraint 1),<br>
          -3x<sub>1</sub> + 3x<sub>2</sub> + x<sub>3</sub> - x<sub>4</sub> - 2x<sub>5</sub> + x<sub>8</sub> = 6 (Constraint 2),<br>
          -x<sub>3</sub> + x<sub>4</sub> + x<sub>6</sub> + x<sub>10</sub> = 1 (Constraint 3),<br>
          x<sub>1</sub> - x<sub>2</sub> + x<sub>3</sub> - x<sub>4</sub> - x<sub>5</sub> + x<sub>11</sub> = 0 (Constraint 4),<br>
          x<sub>j</sub> ≥ 0 (j = 1, 2, ..., 11).
        </div>
      </td>
    </tr>
  </table><br>
  We will now solve this through the simplex method. First list all equations in tableau form, as shown in the initial tableau:
  <img width="834" height="210" alt="image" src="https://github.com/user-attachments/assets/8fd34d20-af5e-4b8c-9880-77938345195d" />
  From step 2, not all objective coefficient is negative, so move on to step 3. To <b>choose the nonbasic variable to pivot in,</b> we choose the <b>largest positive coefficient in the w equation,</b> which is the <b>x<sub>6</sub> column</b> with a coefficient of "3" in the w equation. This is denoted by an <b>arrow pointing into the column at the bottom of the table.</b><br><br>
  On the other hand, to <b>choose the basic variable to leave the basis,</b> we use the <b>ratio test</b>, which in this case is the <b>constraint row with x<sub>10</sub></b> as the basic variable. This is denoted by an <b>arrow pointing out of the row at the left of the table.</b> Note that certain ratios are not computed as they are either division by 0 (giving infinity) or division of negative number (as seen in tableau 3).<br><br>
  The <b>exact entry in table</b> that will <b>become the new basic variable</b> (intersection between the entry column and exit row) is <b>circled.</b><br><br>
  <img width="834" height="210" alt="image" src="https://github.com/user-attachments/assets/0caef0c9-739d-44ec-bf2c-d592f544bee2" />
  Now we have to convert the equations so x<sub>6</sub> becomes the basic variable for constraint row 3, which is <b>step 5 of the algorithm.</b> The procedure is:
  <ol>
    <li>Relabel all the equations (in this case 1-6 is replaced with new labels 7-12).</li><br>
    <li>Perform <b>transformation on the constraint</b> containing the <b>circled entry</b> so that the <b>coefficient of the new basic variable is "1"</b> (in this case no transformation was done as the coefficient was already "1").</li><br>
    <li><b>Update the basic variable entry</b> of that constraint to the new basic variable (in this case x<sub>10</sub> became x<sub>6</sub>).</li><br>
    <li>Perform <b>transformation on other constraints</b> so that the <b>coefficient of the new basic variable is "0"</b> (in this case no transformation was performed on equation 8 and 9 as the coefficient of x<sub>6</sub> was already "0").</li><br>
    A good practice is to use the <b>original equation to add or subtract a multiple of the new equation containing the new basic variable</b> (new equation = original equation ± constant * new constraint with the basic variable). In tableau 2 we can see that equation 7 = equation 1 - 2 * equation 3, but it is better to write equation 9 in place of 3. 
  </ol>
  After updating all the equations, go back to step 1 of the algorithm and repeat the whole thing.<br><br>
  <img width="823" height="186" alt="image" src="https://github.com/user-attachments/assets/72334433-6b63-440f-9576-e73a7cb4c64c" /><br>
  <img width="815" height="188" alt="image" src="https://github.com/user-attachments/assets/051bfeff-9278-40a5-bd05-2165233935bc" /><br>
  At tableau 4, we can see that all <b>artifical variables have been replaced, and w = 0, so the solution is feasible.</b> Now we can <b>end phase 1, drop the w equation,</b> and continue the algorithm (in other words, start phase 2).<br>
  <img width="766" height="175" alt="image" src="https://github.com/user-attachments/assets/e481e331-8be6-45dd-8a8e-41860f6c3c4b" />
  In the final tableau, we have achieved <b>optimal solution</b> as all the <b>coefficients in the objective function is < 0.</b> This means that we will exit in step 2 of the algorithm, giving us:
  <ul>
    <li>x<sub>2</sub> = 2</li>
    <li>x<sub>3</sub> = 4</li>
    <li>x<sub>5</sub> = 2</li>
    <li>x<sub>6</sub> = 5</li>
    <li>max z = 32</li>
  </ul>
