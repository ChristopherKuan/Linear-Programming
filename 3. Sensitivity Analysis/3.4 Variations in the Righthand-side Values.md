### Example (Changing of constraint 3)
<div align='justify'>
  Let us take a closer look at constraint 3, demand for glass x<sub>1</sub>. If I were to change the initial RHS of this constraint by a small amount, we would get:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        $$x_1 + x_6 = 8 + \Delta b_3$$
      </td>
    </tr>
  </table>
  Since x<sub>6</sub>, the <b>slack in this constraint</b>, is a <b>basic variable in the final tableau</b>, x<sub>6</sub> is merely <b>increased or decreased</b> by the small change. Which can be viewed as replacing x<sub>6</sub> as:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        $$x_6 \rightarrow x_6 + \Delta b_3$$
      </td>
    </tr>
  </table>
  And substituting it into the original constraint 3, <b>similar to how replacement seen in 3.1 Shadow Prices work,</b> when we increased production capacity from 60 to 61.<br><br>
  You might be wondering x<sub>1</sub> is a basic variable in the final tableau too, but <b>why is it not changing?</b> It is because it is not the slack variable of this equation, so any changes made will be absorbed by the slack variable.<br><br>
  Now, to <b>keep current solution feasible, x<sub>6</sub> ≥ 0.</b>The current value of x<sub>6</sub> in the final tableau tells us how much is x<sub>6</sub> currently at, meaning that the <b>small change applied to this value must make x<sub>6</sub> greater than or equal to 0:</b><br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        $$x_6 = 1 \frac{4}{7} + \Delta b_3 \geq 0$$
      </td>
    </tr>
  </table>
  Implying that:
  <table align='center'>
    <tr align='center'>
      <td>
        $$\Delta b_3 \geq -1 \frac{4}{7}$$
      </td>
    </tr>
  </table>
  Or equivalently:
  <table align='center'>
    <tr align='center'>
      <td>
        $$b_3^{new} = 8 + \Delta b_3 \geq 6 \frac{3}{7}$$
      </td>
    </tr>
  </table>
  Since the shadow price of x<sub>6</sub> is 0, changes in this constraint (within its allowable range) <b>will not affect the optimal value</b> of the objective function, although the values of the decision variables may still change, so the tableau values would still need to be updated if we want the new solution.
</div>

### Example (Changing of constraint 2)
<div align='justify'>
   Let us take a closer look at constraint 2, warehouse capacity. If I were to change the initial RHS of this constraint by a small amount, we would get:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        $$10 x_1 + 20x_2 + 10 x_3 + x_5 = 150 + \Delta b_2$$
      </td>
    </tr>
  </table>
  With the same reasoning, the <b>change applied affects x<sub>5</sub>,</b> the slack variable. In this case, the substitution applied means that x<sub>5</sub>, which was originally 0 (nonbasic) in the final solution, is now the negative of the small change:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        $$x_5 \rightarrow x_5 + \Delta b_2 = 0$$ in final tableau, giving
      </td>
    </tr>
    <tr align='center'>
      <td>
        $$x_5 = - \Delta b_2$$
      </td>
    </tr>
  </table>
  This also makes the rows in the final tableau become:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>$$x_2 - \frac{3}{35} \Delta b_2 = 4 \frac{2}{7}$$</td>
      <td>Row 1, x<sub>3</sub> and x<sub>4</sub> are nonbasic so = 0</td>
    </tr>
    <tr align='center'>
      <td>$$x_6 - \frac{1}{14} \Delta b_2 = 1 \frac{4}{7}$$</td>
      <td>Row 2, x<sub>3</sub> and x<sub>4</sub> are nonbasic so = 0</td>
    </tr>
    <tr align='center'>
      <td>$$x_1 + \frac{1}{14} \Delta b_2 = 6 \frac{3}{7}$$</td>
      <td>Row 3, x<sub>3</sub> and x<sub>4</sub> are nonbasic so = 0</td>
    </tr>
  </table>
  In order for current basis to remain optimal, the basic variables have to be ≥ 0.<br><br>
  <table align='center'>
    <tr align='center'>
      <td>$$x_2 = 4 \frac{2}{7} + \frac{3}{35} \Delta b_2 \geq 0$$</td>
      <td>that is, $$\Delta b_2 \geq -50$$</td>
    </tr>
    <tr align='center'>
      <td>$$x_6 = 1 \frac{4}{7} + \frac{1}{14} \Delta b_2 \geq 0$$</td>
      <td>that is, $$\Delta b_2 \geq -22$$</td>
    </tr>
    <tr align='center'>
      <td>$$x_1 = 6 \frac{3}{7} - \frac{1}{14} \Delta b_2 \geq 0$$</td>
      <td>that is, $$\Delta b_2 \leq 90$$</td>
    </tr>
  </table>
  Implying that:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        $$-22 \leq \Delta b_2 \leq 90$$
      </td>
    </tr>
  </table>
  Or equivalently,<br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        $$b_2^{new} = 150 + \Delta b_2$$, giving
      </td>
    </tr>
    <tr align='center'>
      <td>
        $$128 \leq b_2^{new} \leq 240$$
      </td>
    </tr>
  </table>
  Since the shadow price of x<sub>5</sub> is nonzero, changes in this constraint (within its allowable range) will affect the optimal value of the objective function, and will also change the values of the decision variables, so the tableau values must be updated. Giving the new final tableau as:<br><br>
  <table align='center'>
    <tr align='center'>
      <th>Basic variables</th>
      <th>Current values</th>
      <th>x<sub>1</sub></th>
      <th>x<sub>2</sub></th>
      <th>x<sub>3</sub></th>
      <th>x<sub>4</sub></th>
      <th>x<sub>5</sub></th>
      <th>x<sub>6</sub></th>
    </tr>
    <tr align='center'>
      <td>x<sub>2</sub></td>
      <td>$$4 \frac{2}{7} + \frac{3}{35} \Delta b_2$$</td>
      <td></td>
      <td>1</td>
      <td>$$- \frac{2}{7}$$</td>
      <td>$$- \frac{1}{7}$$</td>
      <td>$$\frac{3}{35}$$</td>
      <td></td>
    </tr>
    <tr align='center'>
      <td>x<sub>6</sub></td>

    </tr>
    <tr align='center'>

## 3.4.2 General formulation
### Change in RHS
<div align='justify'>
  Consider the <b>initial canonical form:</b><br><br>
  <table align='center'>
    <tr align='center'>
      <td>a<sub>11</sub>x<sub>1</sub></td>
      <td>+a<sub>12</sub>x<sub>2</sub></td>
      <td>+...</td>
      <td>+a<sub>1n</sub>x<sub>n</sub></td>
      <td>+x<sub>n+1</sub></td>
      <td></td>
      <td></td>
      <td></td>
      <td>=b<sub>1</sub></td>
    </tr>
    <tr align='center'>
      <td>a<sub>21</sub>x<sub>1</sub></td>
      <td>+a<sub>22</sub>x<sub>2</sub></td>
      <td>+...</td>
      <td>+a<sub>2n</sub>x<sub>n</sub></td>
      <td></td>
      <td>+x<sub>n+2</sub></td>
      <td></td>
      <td></td>
      <td>=b<sub>2</sub></td>
    </tr>
    <tr align='center'>
      <td>...</td>
      <td></td>
      <td></td>
      <td></td>
      <td></td>
      <td></td>
      <td></td>
      <td></td>
      <td>...</td>
    </tr>
    <tr align='center'>
      <td>a<sub>m1</sub>x<sub>1</sub></td>
      <td>+a<sub>m2</sub>x<sub>2</sub></td>
      <td>+...</td>
      <td>+a<sub>mn</sub>x<sub>n</sub></td>
      <td></td>
      <td></td>
      <td>...</td>
      <td>+x<sub>n+m</sub></td>
      <td>=b<sub>m</sub></td>
    </tr>
  </table>
  is transformed into a <b>canonical form:</b><br><br>
  <table align='center'>
    <tr align='center'>
      <td>$$\bar{a}_{11} x_1$$</td>
      <td>$$+ \bar{a}_{12} x_2$$</td>
      <td>+...</td>
      <td>$$+ \bar{a}_{1n} x_n$$</td>
      <td>$$+ \beta_{11} x_{n+1}$$</td>
      <td>$$+ \beta_{12} x_{n+2}$$</td>
      <td>+...+</td>
      <td>$$+ \beta_{1m} x_{n+m}$$</td>
      <td>$$\bar{b}_1$$</td>
    </tr>
    <tr align='center'>
      <td>$$\bar{a}_{21} x_1$$</td>
      <td>$$+ \bar{a}_{22} x_2$$</td>
      <td>+...</td>
      <td>$$+ \bar{a}_{2n} x_n$$</td>
      <td>$$+ \beta_{21} x_{n+1}$$</td>
      <td>$$+ \beta_{22} x_{n+2}$$</td>
      <td>+...+</td>
      <td>$$+ \beta_{2m} x_{n+m}$$</td>
      <td>$$\bar{b}_2$$</td>
    </tr>
    <tr align='center'>
      <td>...</td>
      <td></td>
      <td></td>
      <td></td>
      <td></td>
      <td></td>
      <td></td>
      <td></td>
      <td>...</td>
    </tr>
    <tr align='center'>
      <td>$$\bar{a}_{m1} x_1$$</td>
      <td>$$+ \bar{a}_{m2} x_2$$</td>
      <td>+...</td>
      <td>$$+ \bar{a}_{mn} x_n$$</td>
      <td>$$+ \beta_{m1} x_{n+1}$$</td>
      <td>$$+ \beta_{m2} x_{n+2}$$</td>
      <td>+...+</td>
      <td>$$+ \beta_{mn} x_{n+m}$$</td>
      <td>$$\bar{b}_m$$</td>
    </tr>
  </table>
  The coefficient b<sub>k</sub> of the k<sub>th</sub> constraint in the <b>initial tableau</b> can be changed by simply substituting:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        $$x_{n+k} \rightarrow x_{n+k} - \Delta b_k$$
      </td>
    </tr>
  </table>
  in the <b>initial tableau,</b> where x<sub>n+k</sub> is the slack/artificial variable associated with k<sub>th</sub> constraint<br><br>
  So in the <b>example of changing constraint 2,</b> n = 3 and k = 2, which is why we were looking at x<sub>5</sub>.<br><br>
  To see how this change <b>affects the updated RHS,</b> make the same substitution in the <b>final tableau.</b> Only the <b>slack/artificial variable terms</b> are affected. This means that:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        Only terms $$\beta_{ik} x_{n+k}$$ for i = 1, 2, ..., m change in the final tableau
      </td>
    </tr>
    <tr align='center'>
      <td>
        They become $$\beta_{ik} (x_{n+k} - \Delta b_k) = \beta_{ik} x_{n+k} - \beta_{ik} \Delta b_k$$
      </td>
    </tr>
  </table>
  Since β<sub>ik</sub> x change in b<sub>k</sub> is a constant, we can move to RHS to give modified RHS values:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        $$\bar{b}_i + \beta_{ik} \Delta b_k$$ (i = 1, 2, ..., m)
      </td>
    </tr>
  </table>
  So in the <b>example of changing constraint 2,</b> the new final current values for each constraints became the sum of old final current value and the change multuplying the coefficient of x<sub>n+k</sub> in that constraint.This is shown by this table<br><br>
  <table align='center'>
    <tr align='center'>
      <td>$$x_2 - \frac{3}{35} \Delta b_2 = 4 \frac{2}{7}$$</td>
      <td>Row 1, x<sub>3</sub> and x<sub>4</sub> are nonbasic so = 0</td>
    </tr>
    <tr align='center'>
      <td>$$x_6 - \frac{1}{14} \Delta b_2 = 1 \frac{4}{7}$$</td>
      <td>Row 2, x<sub>3</sub> and x<sub>4</sub> are nonbasic so = 0</td>
    </tr>
    <tr align='center'>
      <td>$$x_1 + \frac{1}{14} \Delta b_2 = 6 \frac{3}{7}$$</td>
      <td>Row 3, x<sub>3</sub> and x<sub>4</sub> are nonbasic so = 0</td>
    </tr>
  </table>
  As long as <b>these values are nonnegative,</b> the <b>basis specified</b> by the final tableau <b>remains optimal,</b> as the reduced costs have not been changed. This means that <b>the change of coefficient b<sub>k</sub> of the k<sub>th</sub> constraint</b> must be:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        $$Max_i \left\{ \left. \frac{- \bar{b}_i}{\beta_{ik}} \right| \beta_{ik} \gt 0 \right\} \leq \Delta b_k \leq Min_i \left\{ \left. \frac{- \bar{b}_i}{\beta_{ik}} \right| \beta_{ik} \lt 0 \right\}$$
      </td>
    </tr>
  </table>
  In the <b>example of changing constraint 2,</b> it is basically solving the inequality of this table:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>$$x_2 = 4 \frac{2}{7} + \frac{3}{35} \Delta b_2 \geq 0$$</td>
      <td>that is, $$\Delta b_2 \geq -50$$</td>
    </tr>
    <tr align='center'>
      <td>$$x_6 = 1 \frac{4}{7} + \frac{1}{14} \Delta b_2 \geq 0$$</td>
      <td>that is, $$\Delta b_2 \geq -22$$</td>
    </tr>
    <tr align='center'>
      <td>$$x_1 = 6 \frac{3}{7} - \frac{1}{14} \Delta b_2 \geq 0$$</td>
      <td>that is, $$\Delta b_2 \leq 90$$</td>
    </tr>
  </table>
  where you get max(-50, -22) and min(90).<br><br>
  Note that the <b>lower bound disappears if all β<sub>ik</sub> (coefficient of x<sub>n+k</sub> in all constraints) ≤ 0</b>, and <b>upper bound disappears if β<sub>ik</sub> ≥ 0.</b><br><br>
</div>

### Variable transitions
<div align='justify'>
  When the <b>change reaches its upper or lower bound,</b> one of the updated RHS values <b>becomes negative</b>, so that <b>basic variable</b> in <b>row r</b> has to <b>leave the basis.</b><br><br>
  Since we are <b>pivoting in row r,</b> we subtract a multiple of t of the pivot row (row r) from the objective equation, the <b>new objective equation</b> has coefficients:<br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        $$\bar{c}_j - t \bar{a}_{rj}$$, 
      </td>
    </tr>
    <tr>
      <td>
        where
        <ul>
          <li>$$\bar{c}_j$$ is the original objective coefficient of x<sub>j</sub> in the final tableau</li>
          <li>t is the multiple</li>
          <li>$$\bar{a}_{rj}$$ is the coefficient of x<sub>j</sub> of constraint r in the final tableau</li>
        </ul>
      </td>
    </tr>
  </table>
  This can be seen in 3.4.1 Variable transitions, when t was applied to the final tableau.<br><br>
  For new basis to be optimal, <b>each of these coefficients must be nonpositive.</b> Since the coefficient for the <b>basic variable being dropped</b> and its coefficient in the constraint r is 1, we must have t ≥ 0 (shown as coefficient of x<sub>6</sub> in 3.4.1 Variable transitions). This is the <b>lower bound for t.</b><br><br>
  The other <b>coefficients remains nonpositive if a<sub>rj</sub> ≥ 0</b> (for e.g. coefficient of x<sub>5</sub> in 3.4.1 Variable transitions, where it gave t ≥ -2/5, which is lower than). For the remaining ones, it sets the <b>upper bound for t.</b><br><br>
  <table align='center'>
    <tr align='center'>
      <td>
        $$t = Min_j \left\{ \left. \frac{\bar{c}_j}{\bar{a}_{rj}} \right| \bar{a}_{rj} \lt 0 \right\}$$
      </td>
    </tr>
  </table>
  The resulting x<sub>j</sub> giving the ipper bound will be entring the constraint and becoming the <b>new basic variable.</b>
</div>

## 3.4.3 Conclusion
<div align='justify'> 
  In conclusion, variations in the right-hand-side (RHS) values provide insight into how changes in resource availability affect the feasibility and optimality of a linear programming solution while preserving the current basis. By adjusting the RHS through the associated slack or artificial variables, the impact of such changes can be directly analyzed using the final tableau. As long as all basic variables remain nonnegative, the current basis—and therefore the shadow prices and reduced costs—remain valid, even though the values of the decision variables and possibly the objective function may change. The allowable range of RHS variation is determined by maintaining feasibility, and once this range is exceeded, a basic variable becomes negative and must leave the basis, triggering a pivot and a change in the optimal solution structure. Thus, RHS sensitivity analysis not only quantifies how much resources can vary without altering the underlying economic interpretation, but also identifies when and how the optimal solution will adjust beyond these limits.
</div>
