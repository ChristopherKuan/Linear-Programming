# 3.1 Shadow Prices
<div align='justify'>
  Shadow price of a constraint, also known as <i>dual variables, marginal values, or pi values,</i> has the following definition,<br><br>
  <table align='center'>
    <tr>
      <td align='justify'>
        <div align='center'>
          Shadow price
        </div>
        The <i>shadow price</i> associated with a particular constraint is the <b>change in the optimal value of the objective function per unit increase in the righthand-size value for that constraint,</b> all other program data remaining unchanged.
      </td>
    </tr>
  </table>
  Shadow price is only valid <b>within the allowable increase/decrease range</b> of that constraint's RHS. If the change exceeds this range, the current optimal basis may change, and the <b>shadow price will no longer apply.</b><br><br>
  Consider the following linear program:<br><br>
  <table align='center'>
    <tr>
      <td align='center'>
        Maximize z = 500x<sub>1</sub> + 450x<sub>2</sub>
        <div align='left'>
          subject to:
        </div>
        6x<sub>1</sub> + 5x<sub>2</sub> ≤ 60 (Production hours)<br>
        10x<sub>1</sub> + 20x<sub>2</sub> ≤ 150 (hundred sq. ft. storage)<br>
        x<sub>1</sub> ≤ 8 (sales limit of glass A)<br>
        x<sub>1</sub> ≥ 0, x<sub>2</sub> ≥ 0<br>
      </td>
    </tr>
  </table>
  When we initially solve this linear program, it gives a maximum value of:<br><br>
  <ul>
    <li>z = 5142.857, at</li>
    <li>x<sub>1</sub> = 6.429</li>
    <li>x<sub>2</sub> = 4.286</li>
  </ul>
  Now what if we <b>increase the production hours from 60 to 61?</b> This causes a <b>unit increase</b> in the <b>RHS of a constraint,</b> and the resulting <b>change in the value</b> of the <b>objective function</b> is the <b>shadow price of that constraint.</b><br><br>
  To find the new value of objective function without solving the whole linear program again, we can simply solve<br><br>
  <div align='center'>
    6x<sub>1</sub> + 5x<sub>2</sub> = 61<br>
    10x<sub>1</sub> + 20x<sub>2</sub> = 150
  </div><br>
  to find a new optimal solution, as the constraints on production capacity and storage capacity remain binding with this increase. Solving this gives:<br><br>
  <ul>
    <li>x<sub>1</sub> = 6.714</li>
    <li>x<sub>2</sub> = 4.143, giving new z value of</li>
    <li>z = 5221.429</li>
  </ul>
  The increase in z is 5221.429 - 5142.857 = 78.572, so the <b>shadow price associated with production capacity is $78.572 per additional hour of production time.</b><br><br>
  This also means that it is profitable to invest up to $78.572 each week to increase production time by one hour.
</div>

## 3.1 Determining shadow prices
<div align='justify'>
  In this section, we will discuss how to find shadow prices for a general linear program.<br> Consider the following fact:<br><br>
  <table align='center'>
    <tr>
      <td align='justify'>
        The <b>shadow price</b> for a <b>particular constraint</b> is the <b>negative of the coefficient</b> of the <b>appropriate slack (or artificial) variable in the objective function</b> of the <b>final tableau.</b>
      </td>
    </tr>
  </table>
  Now consider the following example to prove this fact:
</div>

## 3.2 Conclusion
<div align='justify'>
  In conclusion, shadow price is the <b>change in the optimal value of the objective function per unit increase in the righthand-size value for that constraint,</b> all other program data remaining unchanged. <br><br>
  It is merely the <b>negative of the coefficient</b> of the appropriate slack (or artificial) variable <b>in the objective function of the final tableau.</b> However, it should be understood that the shadow prices are <b>associated with the constraints and not the variables.</b> Meaning that in the example, the shadow price of 11/14 is for the <b>production hour constraint, not x<sub>4</sub>.</b>
</div>
