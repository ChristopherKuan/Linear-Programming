# 1.1 Results in Linear Programming
<div align="justify">
    In linear programming, a <b>feasible solution</b> is any solution is allowed, meaning that the solution:
    <ul>
        <li>satisfies <b>all constraints</b></li>
        <li>respects <b>non-negativity</b></li>
    </ul>
    On the other hand, an <b>optimal solution</b> is the best among all feasible solutions, meaning that the solution is:
    <ul>
        <li><b>feasible</b>, and</li>
        <li>gives the <b>best value</b> of the <b>objective function</b></li>
    </ul>
</div>

## 1.1.1 Errors in solutions
<div align='justify'>
    Although the simplex method will give the optimal solution, if the <b>problem is poorly formulated</b>, or the <b>requirements exceed the capacity</b> of the existing available resources, the linear program could have <b>no feasible solution.</b> The errors usually met are:
<br><br>
    <ol>
        <li>
            Infeasible solution<br><br>
            It happens when there is <b>no point that satisfies all the constraints</b> at the same time. It may occur due to:<br><br>
            <ul>
                <li>Conflicting constraints (x≥5 and x≤3, nothing can satisfy both)
                <li>Overly restrictive constraints (x+y≤2, x≥2 and y≥2, each constraint is fine individually but they are too tight together)
                <li>Modeling mistakes (wrong inequality signs, coefficients and missing units consistency)
                <li>Logical contradictions in real-world meaning
                <li>Equality constraints that cannot be satisfied
            </ul>
        </li>
        <br>
        <li>
            Unbounded Objective Function<br><br>
            It means there are feasible solutions, but <b>no optimal solution.</b> The <b>objective value can increase (or decrease) forever without violating any constraints</b>. It may occur due to:<br><br>
            <ul>
                <li>Missing constraints in a direction (Maximizing z = x + y, subjected to x≥0, y≥0, meaning can keep increasing x,y → Z → ∞)
                <li>Constraints do not limit the objective (x-y≤10, x,y≥0, so you can increase both x and y together and still satisfy constraint)
                <li>Modeling mistake (forgetting an upper bound like capacity limit, missing resource constraint)
            </ul>
        </li>
    </ol>
</div>

## 1.1.2 Conclusion
<div align='justify'>
    In conclusion, solving a linear program can result in three possible solutions:
    <ol>
        <li><b>Infeasible result,</b> where no values of the decision variables x<sub>1</sub>, x<sub>2</sub>, ..., x<sub>n</sub> simultaneously satisfies all constraints and nonnegativity constraints</li>
        <li><b>Unbounded solution,</b> where in maximizing, the value of objective function can be increased indefinitely without any constraints, and vice versa.</li>
        <li>At least one finite <b>optimal solution</b> or multiple optimal solutions</li>
    </ol>
</div>
